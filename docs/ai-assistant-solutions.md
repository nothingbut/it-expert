## 方案1：应用内智能导航

### 场景描述

在应用程序中嵌入轻量级 AI 助手，实现：
- 理解用户自然语言输入
- 识别用户意图
- 映射到应用功能/路由
- 快速响应（<200ms）

### 技术架构

```
[用户输入] "我想看订单"
    ↓
[本地 AI 模型] (Qwen2.5-0.5B via oMLX API)
    ↓
[意图识别] → "查看订单"
    ↓
[路由映射] → /orders
    ↓
[跳转执行]
```

### 推荐模型

| 模型 | 大小 | 内存 | 速度 | 准确率 | 推荐度 |
|------|------|------|------|--------|--------|
| **Qwen2.5-0.5B** | 1GB | 1.2GB | 50ms | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| SmolLM2-360M | 360MB | 500MB | 30ms | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| BERT分类 | 400MB | 500MB | <10ms | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 传统ML | <10MB | <100MB | <1ms | ⭐⭐⭐⭐ | ⭐⭐⭐ |

### 实现方式

#### 方案A：调用 oMLX API（推荐）⭐⭐⭐⭐⭐

**优势**：
- ✅ 应用轻量（无额外体积）
- ✅ 模型独立部署（易维护）
- ✅ 多应用共享
- ✅ 响应快速（50-100ms）

**架构**：
```
[应用A] ─┐
[应用B] ─┼─→ [oMLX Server:8000] → [Qwen2.5-0.5B]
[应用C] ─┘
```

**关键代码**：
```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="2348"
)

response = client.chat.completions.create(
    model="Qwen2.5-0.5B-Instruct",
    messages=[
        {"role": "system", "content": "你是导航助手"},
        {"role": "user", "content": "我想看订单"}
    ],
    max_tokens=20,
    temperature=0.1
)
```

#### 方案B：Function Calling（更可靠）⭐⭐⭐⭐⭐

**优势**：
- ✅ 结构化输出
- ✅ 类型安全
- ✅ 更高准确率

**示例**：
```python
tools = [{
    "type": "function",
    "function": {
        "name": "navigate_to",
        "description": "导航到应用功能",
        "parameters": {
            "type": "object",
            "properties": {
                "function": {
                    "type": "string",
                    "enum": ["查看订单", "个人设置", "帮助中心"]
                }
            }
        }
    }
}]

response = client.chat.completions.create(
    model="Qwen2.5-0.5B-Instruct",
    messages=[{"role": "user", "content": "我想看订单"}],
    tools=tools,
    tool_choice="auto"
)
```

### 性能指标

**M4 Mac mini (32GB) 预期**：
- 响应时间：50-100ms（缓存命中）
- 首次响应：100-200ms（冷启动）
- 准确率：95%+（Function Calling）
- 内存占用：+0MB（调用API）

### 适用场景

- ✅ 桌面应用（macOS/Windows/Linux）
- ✅ Web 应用（通过 API）
- ✅ 移动应用（通过局域网 API）
- ✅ 功能数量：5-50 个
- ✅ 用户量：单机或小团队

## 方案2：TXT章节智能识别

### 场景描述

从大量 TXT 格式小说中自动识别和提取章节：
- 识别章节标题（第X章、Chapter X等）
- 提取章节内容
- 导出为结构化数据
- 批量处理

### 技术架构

```
[小说TXT文件]
    ↓
[LLM自动标注] → 100+ 样本标注
    ↓
[人工校验] → 提高质量（可选）
    ↓
[训练CRF模型] → 轻量模型（<10MB）
    ↓
[生产部署] → 快速识别（~10ms/章节）
```

### 方案对比

| 方案 | 大小 | 速度 | 准确率 | 训练 | 推荐度 |
|------|------|------|--------|------|--------|
| **正则+规则** | 0MB | <1ms | 85-95% | ❌ | ⭐⭐⭐⭐⭐ |
| **LLM标注+CRF** | <10MB | ~10ms | 92-97% | ✅ | ⭐⭐⭐⭐⭐ |
| **LLM直接** | 1GB | 50ms | 95%+ | ❌ | ⭐⭐⭐ |

### 推荐方案：混合方案

**第一层：正则过滤**（快速处理80%常规格式）
- 处理标准格式：第X章、Chapter X
- 速度：<1ms
- 准确率：90%+

**第二层：CRF模型**（处理特殊格式）
- 处理非标准格式
- 速度：~10ms
- 准确率：95%+

**第三层：LLM兜底**（可选）
- 处理极特殊情况
- 按需调用

### 工作流程

#### 阶段1：数据准备（1-2小时）

1. **收集样本**（10-100本小说）
2. **LLM自动标注**
   - 使用 Qwen2.5-0.5B
   - 批量处理：10行/次
   - 时间：~10分钟/本
3. **人工校验**（可选）
   - 重点检查标记为章节的行
   - 时间：~5分钟/本

#### 阶段2：模型训练（<1分钟）

1. **特征工程**
   - 文本长度、位置、关键词
   - 上下文信息
   - 分词特征
2. **CRF训练**
   - 算法：LBFGS
   - 迭代次数：100
   - 时间：<1分钟
3. **模型评估**
   - 交叉验证
   - F1分数：0.95+

#### 阶段3：生产部署（持续）

1. **主动学习**
   - 识别不确定样本
   - 自动触发标注
   - 增量训练
2. **性能监控**
   - 准确率跟踪
   - 错误案例收集
3. **模型更新**
   - 定期重训练
   - 版本管理

### 主动学习流程

```
[新小说]
    ↓
[CRF预测] → 计算不确定性
    ↓
[不确定性 > 阈值？]
    ├─ Yes → [LLM标注] → [加入训练集] → [重训练]
    └─ No  → [直接使用预测结果]
```

**触发条件**：
- 平均不确定性 > 0.3
- 章节数量异常（过多/过少）
- 章节长度方差大

**优势**：
- ✅ 减少标注成本（只标注困难样本）
- ✅ 持续提升模型
- ✅ 自动适应新格式

### 性能指标

**数据需求 vs 效果**：

| 标注样本 | F1分数 | 准确率 | 适用场景 |
|---------|--------|--------|---------|
| 10本小说 | 0.85-0.90 | 85-90% | 基础可用 |
| 50本小说 | 0.90-0.95 | 90-95% | 良好 |
| 100本小说 | 0.95+ | 95%+ | 优秀 |

**推理性能（M4 Mac mini）**：
- 处理速度：~10ms/章节
- 内存占用：<100MB
- 批量处理：1000章节/秒

### 适用场景

- ✅ 小说网站章节管理
- ✅ 电子书制作工具
- ✅ 内容分析平台
- ✅ 个人小说整理
- ✅ 批量处理需求
