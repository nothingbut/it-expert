## 方案选择

### 选定方案：方案 A - 统一 Playnite 前端 ⭐

**核心架构**:
- **游戏前端**: Playnite（全屏模式 + 现代主题）
- **模拟器管理**: 通过 Playnite 内置模拟器库 + 手动配置
- **应用整合**: 将浏览器、音乐、视频应用作为"游戏"添加到库中
- **自动启动**: Windows 启动时自动打开 Playnite 全屏模式

**选择理由**:
1. 统一现代界面 + 手柄友好
2. 完全免费，不需要额外购买软件
3. 模拟器支持成熟，可以覆盖需求
4. 单一软件栈，易于维护

**拒绝方案**:
- 方案 B（Playnite + LaunchBox）: LaunchBox Big Box 需要付费 $75
- 方案 C（RetroArch 为主）: RetroArch 界面不够现代化

## 核心技术决策

### 存储方案
```
E:\
├─ Games\                    # 游戏根目录
│   ├─ PC-Games\            # PC 游戏
│   ├─ ROMs\                # 模拟器 ROM（分平台目录）
│   └─ Game-Assets\         # 封面、截图、元数据
├─ Emulators\               # 模拟器安装目录
└─ Playnite\                # Playnite 配置和数据
```

### 推荐模拟器组合

| 平台 | 模拟器 | 说明 |
|------|--------|------|
| FC/SFC/MD/GBA 等经典 | RetroArch | 核心包统一管理 |
| N64 | RetroArch 或 Project64 | |
| PS1 | DuckStation | 增强效果好 |
| PS2 | PCSX2 | 兼容性最好 |
| NGC/Wii | Dolphin | 成熟稳定 |
| 3DS | Citra | 官方推荐 |
| Switch | Ryujinx | 兼容性更好（相比 Yuzu） |
| PS3 | RPCS3 | 实验性，部分游戏可玩 |
| Xbox 360 | Xenia | 实验性 |

### 内存分配方案（64GB 总内存）

```yaml
固定分配（需配置文件限制）:
  Hyper-V (飞牛NAS): 4-8GB
  WSL2 (Ubuntu + OpenClaw): 20GB

动态共享池: 约 36-52GB
  - Windows 系统
  - Playnite 界面
  - 游戏和模拟器进程
  - 后台应用

说明: 只有 Hyper-V 和 WSL2 需要在配置文件中设置上限
     其他应用共享系统内存池，Windows 自动管理
```

### 模拟器磁盘占用预估

**不含游戏 ROM 的情况**:
- 最小配置: 1.5 - 2GB
- **推荐配置**: 2.5 - 3.5GB（所有主流平台）
- 完整配置: 4 - 5GB（包含备选和实验性模拟器）

### 界面配置
- **主题**: Strata Fullscreen（Xbox Series X/S 风格）
- **备选主题**: Harmony (PS5 风格)、ModernUI Fullscreen
- **手柄导航**: 启用
- **快捷键**: Guide + Start 退出全屏

### 自动启动方案
```batch
# 启动脚本: C:\Users\[用户名]\startup-playnite.bat
timeout /t 10 /nobreak
start "" "E:\Playnite\Playnite.FullscreenApp.exe"
```
- 添加到 shell:startup 或任务计划程序
- 延迟 10-15 秒确保系统服务加载完成
